# 项目协作规则

1. 用户完全不懂编程。用中文直接说明结果，少讲术语。
2. 默认优先使用 @superpowers 的工作流。
3. 先明确假设和取舍；优先最简单方案；只做与当前需求直接相关的最小修改。
4. 先定义可验证的完成标准，再实现和验证。
5. 遇到 bug、白屏、没反应、报错，默认先排查原因再修复。
6. 如果用户说“先分析原因和解决方案确认”，那就先分析，不要直接改。
7. UI 组件库优先使用 shadcn，没有就根据 shadcn 的风格修改。
8. 每次推送到 GitHub 时，自动更新版本号 +0.0.1，并同步 `version` 和 `appVersion`。
9. 更新公告保持极简，只写新功能和体验优化，去重，不写调试过程。
10. 禁止调用本地浏览器。
