# 结果窗口极简化 PRD

## 1. 背景

当前任务卡片展开态的结果窗口虽然已经合并为单一区域，但视觉层级仍然偏多：

- 外层卡片
- 结果窗口容器
- 顶部切换条
- 主图区域
- 主图内标签
- 底部缩略图区

用户希望进一步极简化，让这块更接近“一张大图 + 少量必要控制”的呈现方式，而不是一个由多层容器和标签堆出来的组件。

## 2. 目标

- 让结果窗口更接近截图中红框区域的那种直给式展示。
- 降低图片查看区的层级感和组件感。
- 保留必要操作，但尽量隐藏或弱化不重要的视觉元素。

## 3. 非目标

- 本次不改动批次生成逻辑。
- 本次不改动参数菜单结构。
- 本次不改动 Lightbox 的核心行为。
- 本次不调整任务列表整体布局结构。

## 4. 问题定义

当前结果窗口的问题主要是“过度组件化”：

- 顶部切换条存在感过强。
- 主图区域仍像被包在一个内层卡片里。
- 原图 / 结果图标签和尺寸标签使主图显得不够干净。
- 缩略图区和主图之间还有明显分区感。

结果是：

- 主图不够突出。
- 用户先看到的是 UI 组件，而不是图片本身。
- 视觉上不够极简。

## 5. 设计方向

总体方向是：

- 图片优先
- 控件退后
- 标签最少
- 分区更弱

换句话说，结果窗口要从“一个复杂的查看组件”改成“以图片为主、交互附着其上的画布”。

## 6. 方案概述

## 6.1 主图区

主图区作为唯一视觉中心，尽量去组件化。

### 目标状态

- 主图是整个结果窗口最强视觉元素
- 不再有明显“卡片套卡片”的感觉
- 边框、底色、阴影尽量收轻

### 调整建议

- 保留最外层一层轻边框即可
- 去掉主图内部额外的厚容器感
- 主图与整体背景更连贯

## 6.2 顶部控制区

顶部切换条保留，但弱化存在感。

### 当前问题

- 原图 / 结果图切换按钮过于像一级按钮
- 右上角预览按钮单独占位，分散视觉注意力

### 调整方向

- 顶部控制区缩成最轻的一行
- 切换按钮更像状态开关，而不是大按钮
- 右上角操作按钮弱化为轻图标按钮

### 目标效果

- 用户知道可以切换
- 但不会觉得这是一个“工具条”

## 6.3 主图标签

当前主图上还存在：

- `原图`
- `结果图 #1`
- 尺寸标签

本次极简化建议：

- 主图左上角标签默认去掉
- 尺寸标签默认去掉
- 只有在确实必要时再通过 hover 或辅助区展示

### 规则建议

- 默认不在主图上长期悬浮显示文字标签
- 主图保持纯净

## 6.4 缩略图区

缩略图区继续保留，但作为主图的附属，不再像一个独立模块。

### 调整方向

- 缩略图直接贴近主图下方
- 减少额外留白
- 不再给缩略图区单独明显分段感

### 视觉目标

- 看起来像主图下方的胶片带
- 而不是“另一个区域”

## 6.5 原图 / 结果图切换

切换行为不变，但视觉上更轻。

### 保留

- 原图
- 结果图

### 简化

- 不用大面积填充按钮
- 可以更像 segmented control 或轻量 pill

## 7. 详细需求

## 7.1 结果窗口层级

当前结果窗口要从多层结构压成以下层级：

1. 结果窗口外容器
2. 顶部轻量控制
3. 主图
4. 底部缩略图

不再出现：

- 主图内额外明显卡片
- 多重背景块
- 主图上的常驻大标签

## 7.2 控件优先级

按优先级排序：

- 第一优先：图片本身
- 第二优先：原图 / 结果图切换
- 第三优先：批次缩略图
- 第四优先：次要文字信息

这意味着：

- 一切文字和装饰应为图片让路

## 7.3 默认展示内容

默认展示应尽量少：

- 主图
- 最少切换控件
- 缩略图

默认不展示：

- 原图/结果图角标
- 尺寸标签
- 重复性的说明文案

## 7.4 批次多图场景

多图场景下仍要支持：

- 点击缩略图切主图
- 当前选中缩略图有明确高亮

但不应通过额外说明文字增强理解，尽量靠视觉结构本身表达。

## 8. 验收标准

### 必须满足

- 结果窗口整体层级明显减少
- 主图成为绝对视觉中心
- 主图上默认不再长期显示多余标签
- 顶部切换区更轻，不像工具栏
- 缩略图区更像主图附属，不像独立卡片区

### 建议满足

- 用户第一眼看到的是图片，不是 UI 元件
- 结果窗口整体高度和长度感更短、更紧凑
- 在多图场景下仍然清晰易切换

## 9. 设计原则

- 去层级，不去能力
- 去标签，不去可理解性
- 去组件感，保留画布感

## 10. 建议实施顺序

1. 先弱化顶部控制区
2. 再清理主图内部标签
3. 再压缩主图和缩略图区之间的结构层级
4. 最后统一整体留白和边框重量

## 11. 待确认点

请确认下面 3 点：

1. 主图上的 `原图 / 结果图 #1 / 尺寸` 这类标签，是否默认全部去掉。
2. 顶部控制区是否接受进一步缩成更轻的 segmented control，而不是现在这种按钮感。
3. 缩略图区是否接受直接贴近主图，不再保留明显的上下分段留白。
