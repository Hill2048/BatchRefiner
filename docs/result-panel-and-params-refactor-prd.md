# 结果区与参数区整理 PRD

## 1. 背景

当前任务卡片已经支持批次生图，但现阶段界面存在 3 个明显问题：

- 原图区域和结果图区是分开的，视觉上像两个模块，信息重复且占空间。
- 批次参数单独占一块，和比例、分辨率割裂，参数区层级不够清晰。
- 小卡片外显了 `已返回 1/1`，在单张场景下信息冗余，噪音偏高。

本次目标是整理任务卡片的信息架构，让结果区更紧凑，参数区更统一，列表态信息更克制。

## 2. 目标

- 将“原图 + 结果图”整合为一个统一结果窗口。
- 将批次参数收进和比例、分辨率同一层级的二级参数菜单。
- 小卡片仅在多张结果图场景下显示返回图片数，单张不显示。

## 3. 非目标

- 本次不调整批次生图的底层逻辑。
- 本次不修改导出规则。
- 本次不新增新的参数类型。
- 本次不重做 Lightbox 交互逻辑，只调整卡片内入口和信息结构。

## 4. 问题定义

### 4.1 结果区割裂

当前展开态卡片同时存在：

- 顶部原图展示区
- 下方结果图区

导致问题：

- 用户需要来回判断“当前看的到底是源图还是结果图”
- 上下结构过长
- 原图和结果图关系不够直观

### 4.2 参数区层级不统一

当前“专属参数”里：

- 比例 / 分辨率是一个参数组件
- 批次是另一块独立选择器

导致：

- 参数结构显得分散
- 批次和其他生成参数不像同一组设置
- 小屏下尤其拥挤

### 4.3 列表态信息噪音

当前小卡片上会显示：

- `已返回 1/1`

但在单张图场景，这个信息没有实际价值，只会增加视觉负担。

## 5. 方案概述

## 5.1 结果图和原图窗口合并

将展开态任务卡片中的原图展示和结果图展示整合成一个统一的“结果窗口”。

### 新结构

统一结果窗口内包含：

- 主显示区域
- 原图 / 结果图切换或对比入口
- 批次结果图缩略图区

### 展示原则

- 默认主显示区展示当前选中的结果图
- 如果任务有原图，则原图作为同一窗口内的辅助查看对象
- 不再让“原图”和“结果图”作为上下两个独立大模块存在

### 用户感知

用户会感觉成：

- 这是一个任务的主画面窗口
- 我可以在这里看结果图
- 也可以切到原图或进行对比

而不是：

- 上面一块看原图
- 下面一块看结果图

## 5.2 参数区合并为统一二级菜单

将批次参数并入当前比例 / 分辨率所在的参数组件，作为同一套“生成参数”的二级菜单内容。

### 新结构

“专属参数”区域中保留一个统一入口，例如：

- `自动 / 1K / x2`

点击后展开参数面板，面板中包含：

- 比例
- 分辨率
- 批次

### 层级要求

- 批次不是独立悬浮块
- 批次和比例、分辨率同级
- 统一在一个参数弹层或二级菜单里管理

### 展示要求

在折叠态按钮上，可展示当前关键组合信息，例如：

- `自动 / 1K / x1`
- `3:4 / 2K / x4`

如果任务跟随全局，可继续沿用现有“任务级覆盖”的逻辑，但展示方式统一到同一个组件里。

## 5.3 小卡片外显返回图片数优化

调整卡片收起态的信息展示规则：

### 单张结果图

- 不显示 `已返回 1/1`
- 保持卡片简洁

### 多张结果图

- 在卡片外显返回图片数
- 只在有价值时显示，例如：
  - `已返回 2/4`
  - `3 张结果`

### 建议规则

- 如果目标批次为 `x1`，不显示
- 如果目标批次大于 `x1`，显示返回情况

## 6. 详细需求

## 6.1 展开态结果窗口

展开态卡片中的“主内容窗口”应整合原图和结果图。

建议包含：

- 主图区域
- 批次结果缩略图列表
- 原图查看入口
- 对比查看入口

### 主图区域规则

- 默认显示当前选中的结果图
- 点击其他结果缩略图时，主图切换
- 若存在原图，可提供：
  - `查看原图`
  - `原图对比`

### 原图位置规则

- 原图不再单独占据一个完整模块高度
- 原图作为辅助内容，收纳进统一结果窗口

## 6.2 参数二级菜单

当前参数入口升级后，需要支持：

- 比例选择
- 分辨率选择
- 批次选择

### 交互要求

- 用户点击参数入口后，一次能看到三类参数
- 切换批次时，不需要跳到另一个独立区域
- 小屏下也能完整显示，不拥挤

### 文案建议

- `比例`
- `分辨率`
- `批次`

## 6.3 小卡片信息规则

### 必须显示

- 任务标题
- 状态
- 必要参数摘要

### 条件显示

- 多图时显示返回张数
- 单图时不显示返回张数

### 不建议显示

- 单图场景下的 `1/1`

## 7. 交互说明

## 7.1 卡片展开态

用户展开卡片后：

- 首先看到统一结果窗口
- 原图与结果图的关系在同一区域内完成
- 参数区更短、更集中

## 7.2 结果图切换

如果任务有多张结果图：

- 在统一结果窗口中切换不同结果图
- 当前选中的图同步影响预览入口

## 7.3 参数调整

用户点击参数入口后：

- 展开统一参数层
- 在里面改比例、分辨率、批次

## 8. 验收标准

### 必须满足

- 原图和结果图区不再是上下两个独立大模块
- 批次参数被收进与比例、分辨率同一个参数菜单
- 小卡片在单张结果图时不显示返回数量
- 小卡片在多张结果图时显示返回数量
- 整体卡片高度和信息密度优于当前版本

### 建议满足

- 展开态首屏能更聚焦结果图本身
- 参数区更短、更整洁
- 小屏下布局更稳定，不会因为批次区单独存在而拥挤

## 9. 风险

- 合并结果窗口会影响当前展开态卡片的结构，需要同步梳理原图、结果图、缩略图和预览入口。
- 参数组件要从“两块独立逻辑”合并成“一套统一逻辑”，需要避免覆盖关系混乱。
- 如果结果窗口整合不好，可能会让原图入口过深，因此需要保持对比入口可发现。

## 10. 建议实施顺序

1. 先重构任务卡片展开态，将原图和结果图合并成统一窗口。
2. 再重构参数组件，把批次并入比例 / 分辨率的二级菜单。
3. 最后调整小卡片摘要规则，只在多图时显示返回数量。

## 11. 待确认点

请确认下面 3 点：

1. 原图在统一结果窗口里，是否接受改成缩略入口或“对比按钮”形式，而不再单独占一整块区域。
2. 参数入口折叠态是否接受直接显示三段信息，例如 `自动 / 1K / x2`。
3. 小卡片多图数量文案更偏向：
   - `已返回 2/4`
   - `2 张结果`
